#### 冒泡排序

对一个无序的序列进行排序时，每次取当前没有排序的数据与下一个数据进行比较，大于则进行交换，然后让下一个数据成为当前数据，再与下一个数据进行比较交换，直到把当前排序序列最大的数据排到最后的位置上。则最后一个位置的数据是有序的，接着对剩下的序列进行相同的排序方式，直到所有数据都进行了排序。

稳定的算法，时间复杂度是所有排序算法中最差的O\(N^2\)。

```
    public static void BubbleSort(int[] arr) {
        int temp;
        int last = 0;
        int k = arr.length - 1;
        for (int i = 0; i < arr.length; i++) {
            for (int j = 1; j < arr.length - last; j++) {
                if (arr[j] < arr[j - 1]) {
                    temp = arr[j];
                    arr[j] = arr[j - 1];
                    arr[j - 1] = temp;
                    // 记录最后一次交换发生的位置
                    k = j - 1;
                }
            }
            // 获取后面有序序列的长度，减1是因为下标比实际位置小一位
            last = arr.length - k - 1;
        }
    }
```

#### 选择排序

每次找出当前排序序列中最小或最大的值，将它交换到当前序列的第一个位置，直到所有序列的最大或最小值都排序完毕。

冒泡排序比较和交换的时间复杂度都是O\($$N^2$$\)，而选择排序虽然比较时间复杂度也是O\($$N^2$$\)，但是交换的事件复杂度降为O\(N\)。

是稳定的排序算法。

```
    public static void chooseSort(int[] arr) {
        int min;
        int temp;
        for (int i = 0; i < arr.length; i++) {
            min = i;
            for (int j = i + 1; j < arr.length; j++) {
                if (arr[min] > arr[j]) {
                    min = j;
                }
            }
            if (min != i) {
                temp = arr[min];
                arr[min] = arr[i];
                arr[i] = temp;
            }
        }
    }
```

#### 直接插入排序

假设当前序列的前n个数据是有序的，接下来取第n+1个数据与该序列进行比较，插入该序列合适的位置，则有序序列扩展为n+1个，

不断重复该过程，直到序列的剩余所有数据都插入到有序序列中；

比较和交换的次数相比冒泡减少一半，比较操作比选择排序更快，因为不需要遍历每一个元素，只需要比较前n个比当前元素大的元素即可。在数据量巨大时，效率不如选择排序，因为选择排序每次遍历只需要交换一次，而插入排序需要多次错位赋值操作，来腾出合适的插入位置。

时间复杂度为O\(N^2\)，是稳定的排序算法。

```
    public static void insertSort(int[] arr) {
        int temp;
        for (int i = 1; i < arr.length; i++) {
            temp = arr[i];

            int j = i - 1;
            for (; j >= 0; j--) {
                if (arr[j] > temp) {
                    arr[j + 1] = arr[j];
                } else {
                    break;
                }
            }
            // for循环最后还会减去1，所以此处需要补1
            arr[j+1] = temp;
        }
    }
```

#### 希尔排序

希尔排序又叫缩小增量排序，基于直接插入排序。

在直接插入排序中，左边是已经排序好的有序序列，而右边则是待排序的序列；排序时，每次从右边的无序序列取出第一个元素，与左边的有序序列进行比较，然后插入到合适位置上。

希尔排序在直接插入排序的基础上，改变了每次排序元素的间隔，将序列中同一间隔的元素编为一组，对其进行直接插入排序。然后，逐渐缩小排序的间隔，直到最后间隔为1，排序完毕；

希尔排序的时间复杂度受到选用的增量序列影响，具体时间复杂度目前仍然未能够确定，但是可以肯定的是，其速度比O\($$N^2$$\)级别的算法快得多，平均为O\(n√n\)，最坏情况下\(间隔选为N/2\)，才会降为O\($$N^2$$\)。

稳定的排序算法。

```
    public static void hellSort(int[] arr) {
        int n = arr.length / 2;
        int temp;

        while (n > 0) {
            for (int i = n; i < arr.length; i+=n) {
                temp = arr[i];

                int j = i - n;
                for (; j >= 0; j = j - n) {
                    if (arr[j] > temp) {
                        arr[j + n] = arr[j];
                    } else {
                        break;
                    }
                }
                arr[j + n] = temp;
            }
            n/=2;
        }
    }
```

#### 堆排序

堆排序不同于之前的任何算法，它是基于二叉树的特点而创造出来的排序算法。

将待排序的序列当成是有n个结点的二叉树，根据大顶堆的特点，对结点的顺序进行调整，使之符合大顶堆的结构特点。然后输出堆顶元素，得到待排序元素中的最大值。接着，重复上述构造过程，直到所有的元素都输出完毕。

堆排序的过程就是不断构造大顶堆\(或小顶堆\)的过程。

堆排序的时间复杂度为O\(nlogn\)，其中构造堆的时间复杂度为logn，总共循环n-1次。

堆排序是不稳定的排序算法。

#### 快速排序

快速排序基于分治的思想。

从当前的排序的两端开始，向中间遍历，遍历的过程中不断的交换两端的元素，让序列逐渐形成一个中间值在中间，左边的元素都比中间值小，右边都比它大的情况。接着操作就是分治思想所在，以除去中间值的两边的序列为新的序列，采用上面相同的方式再找出两个子序列的中间值，不断重复，直到只剩下一个元素为止。

此处的中间只是在理想状态下能够取到中间值，实际上，我们通过提供第一个，最后一个或者中间的元素为作为基准值，取的基准值在序列中的排序的位置是不确定的，可以保证的是，最终排序后，基准值的左边都是小于它的元素，而右边都是大于它的元素。

时间复杂度为O\(NlogN\)，最坏情况达到O\(N^2\)，不是稳定的排序算法。

```
    public static void quickSort(int[] arr) {
        quickSort(arr, 0, arr.length - 1);
    }

    public static void quickSort(int[] arr, int low, int high) {
        if (low < high) {
            int middle = middle(arr, low, high);
            quickSort(arr, low, middle - 1);
            quickSort(arr, middle + 1, high);
        }
    }

    private static int middle(int[] arr, int low, int high) {
        int temp = arr[low];
        while (low < high) {
            while (low < high && arr[high] >= temp) {
                high--;
            }
            arr[low] = arr[high];

            while (low < high && arr[low] <= temp) {
                low++;
            }
            arr[high] = arr[low];
        }

        arr[low] = temp;
        return low;
    }
```

#### 归并排序

归并排序基于递归分治的思想。

递归分治的反推过程：将n个元素的无序序列分割为n个有序的序列\(只有一个元素的情况下，认为是有序的\)，然后开始两两合并这些有序序列。合并的过程对它们进行排序，使之成为新的有序序列。接着，重复上述步骤，不断的的两两合并为新的有序序列，直到最后只剩下一个有序序列，排序完成。

时间复杂度为O\(NlogN\)，一般认为是稳定的算法。

#### 基数排序

也叫桶排序，是一种只能用在整数上的排序，并且无法同时排序正负两种整数。

* 最低位优先法（适合于位数较少的情况）

首先排序的是基于十进制的整数，所以先创建十个桶；

先从个位开始，遍历的每个元素的个位数，按大小分配到对应的桶中，按顺序从桶中取出所有的元素，组成新的序列，此时的序列已经是按照个位数排序好了；

接着遍历十位，同样按相同的方式，放进对应的桶中，接着再顺序取出来，新的序列已经按照十位的大小排序完成；

接着遍历百位..

同样的方式进行遍历，直到所有数中的最大值的最大位也被排过序，此时取出合成的序列就排序完毕。

需要注意的是，对于不足位数的元素，在对应的位数上进行补0即可。

* 最高位优先法

最高位优先法步骤类似与最低位优先法，不同点在于最高位优先法引入了递归分治的思想。

先从最高位开始排序，当桶中只有一个元素时，则这个元素已经排序完毕。对于元素个数多于1个的桶，再次采用最高位优先法对桶中元素进行排序，直到没有桶的元素个数多于一个，开始合并每个桶中的元素，每一层的最高位优选排序合成都是有序序列，并且都是上一层的子序列，将所有层的序列合并起来，则为最终的有序序列。

基数排序是稳定的排序算法，在最坏情况下，时间复杂度也能达到O\(NlogN\)
